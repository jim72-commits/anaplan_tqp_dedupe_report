# v1.2 — SharePoint folder auto-refresh

The Overlap Report (`index.html` **v1.2**) can poll your SharePoint library folder and re-run **Analyze** when the SRB or Rules CSV changes.

## Why you cannot paste the SharePoint “AllItems.aspx” URL alone

That URL is a **human-facing library page**. Browsers block anonymous cross-origin reads to SharePoint, and listing folder contents requires **Microsoft Graph** with a signed-in user (or a server-side app). The report therefore implements:

1. **Microsoft 365 / Graph (recommended)** — Sign in with your Moody’s work account; the app calls Graph to list files in the folder and download them when timestamps change.
2. **Direct CSV URLs** — If IT publishes two stable HTTPS links (for example via Azure Blob + SAS, or an internal API), the app can poll `HEAD`/`GET` and reload when `ETag` / `Last-Modified` changes.

## Target folder (from your roadmap library)

Default settings in the Sync dialog match:

| Setting | Value |
|--------|--------|
| Site | `moodys.sharepoint.com` / `sites/MA_Roadmap` |
| Document library | `Territory Quota Management` |
| Folder path | `Territory & Quota Planning 2027/External Pages in TQP/1. Overlap Report` |

You must enter the **exact CSV file names** as they appear in that folder (case-insensitive match).

## Azure AD app registration (Graph mode)

Your Entra ID (Azure AD) admin must register a **single-page application**:

1. **Redirect URI**: Must match how users open the report, for example:
   - `https://moodys.sharepoint.com/.../Overlap_Report/index.html` (if hosted in SharePoint), or  
   - `http://127.0.0.1:8765/index.html` (local testing only).
2. **API permissions (delegated)**:
   - `User.Read`
   - `Sites.Read.All`  
   Tenant admin consent is often required for `Sites.Read.All`.
3. Copy **Application (client) ID** and **Directory (tenant) ID** into the Sync dialog.

### Redirect URI checklist

MSAL compares the **full page URL** (including path to `index.html`). Every deployment URL must be registered as a separate redirect URI.

## Using the Sync UI

1. Open the **Overlap Report** tab — **Sync** appears in the header next to import controls.
2. Choose **Microsoft 365 / Graph** or **Direct CSV URLs**.
3. Fill in IDs / file names or URLs → **Save settings**.
4. **Graph**: Click **Sign in** once, then **Check now** to verify.
5. Enable **automatic refresh** to poll on an interval (minimum 2 minutes). Polling runs **only while the browser tab is open** (no background service worker in this version).

**New Files** clears local cache and **stops** auto-polling until you turn it on again in Sync settings.

## Limitations (v1.2)

- **Tab must stay open** for scheduled polling.
- **Graph throttling**: Very short intervals may hit Microsoft Graph limits; default 15 minutes is safe for most cases.
- **File names** must match exactly (extension `.csv`).
- **Rules file** is loaded into memory like a manual upload; very large files follow the same constraints as drag-and-drop.

## Alternative: Power Automate → HTTPS URLs

If Graph consent is slow to obtain, IT can:

1. Trigger on “file modified” in the SharePoint folder.
2. Copy both CSVs to two stable HTTPS endpoints (with SAS or API key).
3. Use **Direct CSV URLs** mode in the Sync dialog (optional Bearer token field).

## Troubleshooting

| Symptom | What to check |
|--------|----------------|
| “Sign in” fails | Redirect URI not registered for this exact URL; pop-up blocked. |
| “Document library not found” | Library display name must match Graph (`Territory Quota Management`). |
| “File not found in folder” | File name typo; file not yet synced to that folder. |
| Graph 403 | `Sites.Read.All` not consented; user lacks access to the library. |
| URL mode CORS error | Host must allow browser `GET` from your report origin (IT may need `Access-Control-Allow-Origin`). |
